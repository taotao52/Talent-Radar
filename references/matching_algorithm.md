# 匹配算法说明

本文档详细说明人才雷达系统的匹配算法原理和实现。

## 算法概述

人才雷达采用**多维度加权匹配算法**，通过5个核心维度对候选人与岗位进行综合评估。

### 核心理念

1. **语义理解优先**：不只做关键词匹配，而是理解候选人的真实能力
2. **可解释性**：每个匹配结果都有详细的匹配理由
3. **动态权重**：不同岗位类型的权重可以调整
4. **双向评估**：同时评估"人适岗"和"岗适人"

## 匹配维度

### 1. 技能匹配（默认权重：35%）

**目标**：评估候选人是否具备岗位所需的技能

**计算方法**：
```
技能匹配分 = 必须技能得分 × 70% + 优先技能得分 × 30%

必须技能得分 = (匹配的必须技能数 / 总必须技能数) × 100
优先技能得分 = (匹配的优先技能数 / 总优先技能数) × 100
```

**匹配规则**：
- 精确匹配：技能名称完全一致
- 模糊匹配：技能名称相似（如"JS"匹配"JavaScript"）
- 语义匹配：技能含义相同（如"容器化"匹配"Docker"）

**评分标准**：
| 匹配情况 | 得分 |
|---------|------|
| 完全匹配（精通） | 100 |
| 完全匹配（熟悉） | 80 |
| 完全匹配（了解） | 60 |
| 相关技能匹配 | 40 |
| 不匹配 | 0 |

### 2. 经验匹配（默认权重：25%）

**目标**：评估候选人的工作经验是否符合岗位要求

**计算方法**：
```
经验匹配分 = 年限得分 × 40% + 行业得分 × 30% + 级别得分 × 30%

年限得分：
  - 达标或超过：100分
  - 达标80%以上：80分
  - 达标60-80%：60分
  - 达标60%以下：40分

行业得分：
  - 完全匹配：100分
  - 相关行业：60分
  - 不匹配：30分

级别得分：
  - 匹配：100分
  - 接近：70分
  - 不匹配：40分
```

### 3. 教育匹配（默认权重：15%）

**目标**：评估候选人的教育背景是否符合岗位要求

**计算方法**：
```
教育匹配分 = 学历得分 × 60% + 专业得分 × 40%

学历等级映射：
  - 大专/专科：1
  - 本科/学士：2
  - 硕士/研究生：3
  - 博士：4

学历得分：
  - 达标或超过：100分
  - 低一级：70分
  - 低两级：40分

专业得分：
  - 完全对口：100分
  - 相关专业：70分
  - 不相关：40分
```

### 4. 文化匹配（默认权重：15%）

**目标**：评估候选人的工作风格和价值观是否与团队匹配

**计算方法**：
```
文化匹配分 = 基础分 + 软技能加分 + 稳定性加分

基础分：70分

软技能加分（每项+5分，上限30分）：
  - 沟通能力
  - 团队协作
  - 学习能力
  - 抗压能力
  - 责任心
  - 自驱力

稳定性加分：
  - 平均工作时长≥2年：+10分
  - 平均工作时长1-2年：+5分
  - 平均工作时长<1年：0分
```

### 5. 发展潜力（默认权重：10%）

**目标**：评估候选人的成长空间和学习能力

**计算方法**：
```
发展潜力分 = 基础分 + 学习能力加分 + 技能广度加分 + 项目经验加分 + 教育背景加分

基础分：70分

学习能力加分：
  - 有证书/持续学习记录：+15分

技能广度加分：
  - 掌握5个以上技能：+10分
  - 掌握3-5个技能：+5分

项目经验加分：
  - 3个以上项目：+10分
  - 1-3个项目：+5分

教育背景加分：
  - 知名院校（985/211）：+10分
```

## 总分计算

```
总匹配度 = Σ(维度得分 × 维度权重)

示例：
假设各维度得分：
  - 技能匹配：80分
  - 经验匹配：70分
  - 教育匹配：90分
  - 文化匹配：85分
  - 发展潜力：75分

总匹配度 = 80×0.35 + 70×0.25 + 90×0.15 + 85×0.15 + 75×0.10
         = 28 + 17.5 + 13.5 + 12.75 + 7.5
         = 79.25分
```

## 匹配等级划分

| 匹配度 | 等级 | 说明 | 建议 |
|-------|------|------|------|
| 90-100 | 完美匹配 | 各方面都非常符合 | 强烈推荐，优先安排面试 |
| 80-89 | 高度匹配 | 大部分要求都满足 | 推荐，可直接进入面试 |
| 70-79 | 良好匹配 | 基本满足要求 | 可考虑，需关注短板 |
| 60-69 | 基本匹配 | 部分要求满足 | 谨慎考虑，需培训或调整 |
| <60 | 匹配不足 | 与要求差距较大 | 不推荐，差距较大 |

## 权重调整机制

### 默认权重

```python
default_weights = {
    'skill': 0.35,      # 技能匹配
    'experience': 0.25, # 经验匹配
    'education': 0.15,  # 教育匹配
    'culture': 0.15,    # 文化匹配
    'potential': 0.10   # 发展潜力
}
```

### 按岗位类型调整

| 岗位类型 | 技能 | 经验 | 教育 | 文化 | 潜力 |
|---------|------|------|------|------|------|
| 技术岗 | 40% | 25% | 15% | 10% | 10% |
| 管理岗 | 20% | 30% | 15% | 25% | 10% |
| 产品岗 | 25% | 25% | 15% | 20% | 15% |
| 应届生 | 30% | 10% | 25% | 15% | 20% |

### 自定义权重

用户可以根据实际需求调整权重：

```python
matcher = TalentMatcher(weights={
    'skill': 0.40,
    'experience': 0.30,
    'education': 0.10,
    'culture': 0.10,
    'potential': 0.10
})
```

## 匹配算法的局限性

### 当前局限

1. **软技能评估依赖推断**：无法直接测量候选人的软技能
2. **文化匹配较主观**：不同公司文化差异大，难以标准化
3. **发展潜力评估有限**：只能基于已有信息推断
4. **行业知识有限**：某些垂直行业的特殊要求可能遗漏

### 改进方向

1. **引入大模型语义分析**：更准确地理解简历和JD
2. **增加面试评估**：结合面试表现进行综合评估
3. **历史数据学习**：根据录用结果优化匹配算法
4. **行业知识库**：建立各行业的专业技能库

## 使用建议

### 对于HR

1. **不要只看总分**：关注各维度的详细分析
2. **结合实际情况**：匹配结果只是参考，需结合面试表现
3. **关注短板**：候选人的短板是否可以通过培训弥补
4. **动态调整**：根据岗位实际需求调整权重

### 对于求职者

1. **了解差距**：重点关注不达标的维度
2. **针对性提升**：根据差距分析制定学习计划
3. **优化简历**：根据匹配结果优化简历内容
4. **合理定位**：选择匹配度60%以上的岗位投递
