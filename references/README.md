# 人才雷达 - 智能人岗匹配诊断系统

> 基于大模型的智能人岗匹配系统，通过语义理解实现简历与岗位的深度分析和精准匹配。

## ⚠️ 重要声明

### 隐私与数据保护

本工具涉及处理个人敏感信息（简历、求职数据等），使用时请注意：

1. **数据最小化原则**：仅收集和处理与岗位匹配直接相关的必要信息
2. **知情同意**：在处理他人简历前，应获得数据主体的明确同意
3. **数据安全**：妥善保管包含个人信息的数据，处理完成后及时删除
4. **合规使用**：遵守当地数据保护法规（如《个人信息保护法》）

### 决策辅助声明

**本工具仅作为决策辅助参考，不应作为自动化招聘决策的唯一依据：**

- 文化匹配、软技能等维度的评估基于有限的文本分析，可能存在偏差
- 所有招聘决策应由人类审核者最终确认
- 建议结合面试、背景调查等多种评估方式

### 公平性承诺

本工具已移除基于学校名称、性别、年龄等非能力因素的歧视性评分：
- 不基于毕业院校进行加分或减分
- 不基于性别、年龄等受保护特征进行评估
- 仅评估与岗位要求直接相关的技能和经验

## 🎯 项目简介

人才雷达是一个面向开发者大赛的Skill作品，旨在解决人才市场中"人岗匹配"的核心痛点。

**核心价值**：
- 🏢 **企业端**：高效筛选简历、精准评估人才、降低招聘成本
- 👤 **个人端**：诊断求职差距、优化简历内容、规划职业发展

**技术特色**：
- 🧠 语义理解匹配（非简单关键词匹配）
- 📊 多维度加权评估算法
- 📝 可解释的匹配理由
- 🔄 双向匹配（人适岗 + 岗适人）

## 🚀 快速开始

### 环境要求

- Python 3.8+
- 无额外依赖（使用标准库）

### 使用方式

#### 方式一：作为Hermes Skill使用

1. 将 `talent-radar` 目录复制到 `~/.hermes/skills/` 目录
2. 重启Hermes Agent
3. 在对话中触发使用：
   - "帮我分析这份简历是否适合Java高级开发岗位"
   - "我想应聘产品经理，帮我看看简历有什么问题"
   - "帮我找出这10份简历中最适合这个岗位的前3名"

#### 方式二：独立使用Python脚本

```python
from scripts.resume_parser import parse_resume_text, resume_to_dict
from scripts.job_analyzer import parse_job_description, jd_to_dict
from scripts.matcher import TalentMatcher, match_to_dict
from scripts.report_generator import generate_hr_report

# 1. 解析简历
resume_text = """
张三
电话：13800138000
5年Java开发经验
精通Java、Spring Boot、MySQL
"""
resume = parse_resume_text(resume_text)

# 2. 解析岗位JD
jd_text = """
高级Java开发工程师
任职要求：
- 5年以上Java开发经验
- 精通Java、Spring Boot
"""
jd = parse_job_description(jd_text)

# 3. 执行匹配
matcher = TalentMatcher()
result = matcher.match(resume_to_dict(resume), jd_to_dict(jd))

# 4. 生成报告
report = generate_hr_report(match_to_dict(result), resume_to_dict(resume), jd_to_dict(jd))
print(report)
```

## 📊 功能特性

### 1. 简历解析

自动从简历文本中提取结构化信息：

| 提取内容 | 说明 |
|---------|------|
| 基本信息 | 姓名、联系方式、邮箱 |
| 工作年限 | 自动计算总工作年限 |
| 教育经历 | 学校、学历、专业、时间 |
| 工作经历 | 公司、职位、职责、成就 |
| 技能清单 | 技术技能、熟练程度 |
| 项目经验 | 项目名称、角色、技术栈 |
| 证书资质 | PMP、AWS认证等 |

### 2. 岗位分析

自动解析岗位描述（JD）：

| 分析内容 | 说明 |
|---------|------|
| 硬性要求 | 学历、经验、必须技能 |
| 软性要求 | 性格、文化匹配 |
| 隐性要求 | 行业背景、管理经验 |
| 岗位职责 | 主要工作内容 |
| 福利待遇 | 薪资、福利 |

### 3. 智能匹配

多维度加权匹配算法：

| 匹配维度 | 默认权重 | 评估内容 |
|---------|---------|---------|
| 技能匹配 | 35% | 核心技能、辅助技能、技能深度 |
| 经验匹配 | 25% | 年限、行业、岗位级别 |
| 教育匹配 | 15% | 学历、专业、院校 |
| 文化匹配 | 15% | 工作风格、价值观、稳定性 |
| 发展潜力 | 10% | 学习能力、技能广度、项目经验 |

### 4. 报告生成

支持多种报告格式：

- **HR报告**：面向招聘方，包含匹配度、推荐意见、风险提示
- **求职者报告**：面向求职者，包含差距诊断、优化建议、学习路径
- **差距分析报告**：详细的差距清单和提升路径

## 🎨 创新亮点

### 1. 语义理解匹配

不同于简单的关键词匹配，本系统通过大模型理解候选人的真实能力：

```
简历写："负责公司核心交易系统的设计与开发"
岗位要求："有高并发系统经验"

传统匹配：❌ 不匹配（没有"高并发"关键词）
本系统匹配：✅ 匹配（理解"核心交易系统"通常涉及高并发）
```

### 2. 可解释的匹配理由

每个匹配维度都有详细的匹配理由：

```
技能匹配：85分
✅ 满足要求：Java（精通）、Spring Boot（熟练）、MySQL（熟练）
⚠️ 部分满足：Redis（了解，要求熟练）
❌ 缺失技能：Kafka
```

### 3. 双向匹配评估

同时评估"人适岗"和"岗适人"：

- **人适岗**：候选人是否满足岗位要求
- **岗适人**：岗位是否符合候选人的职业发展

### 4. 动态权重调整

不同岗位类型可以调整权重：

```python
# 技术岗：更看重技能
tech_weights = {'skill': 0.40, 'experience': 0.25, ...}

# 管理岗：更看重经验和领导力
mgmt_weights = {'skill': 0.20, 'experience': 0.30, 'culture': 0.25, ...}
```

## 🔧 技术实现

### 核心算法

采用多维度加权匹配算法：

```
总匹配度 = Σ(维度得分 × 维度权重)

维度得分 = 基础分 × 完成度系数 × 质量系数
```

### 数据结构

```python
@dataclass
class MatchResult:
    total_score: float          # 总分
    level: MatchLevel           # 匹配等级
    dimensions: List[DimensionScore]  # 各维度得分
    overall_strengths: List[str]  # 总体优势
    overall_weaknesses: List[str] # 总体劣势
    recommendation: str         # 推荐意见
    match_summary: str          # 匹配摘要
```

### 匹配等级

| 分数范围 | 等级 | 建议 |
|---------|------|------|
| 90-100 | 完美匹配 | 强烈推荐，优先安排面试 |
| 80-89 | 高度匹配 | 推荐，可直接进入面试 |
| 70-79 | 良好匹配 | 可考虑，需关注短板 |
| 60-69 | 基本匹配 | 谨慎考虑，需培训 |
| <60 | 匹配不足 | 不推荐，差距较大 |
